# 어휘 구조 (Lexical Structure)

SQL 입력은 일련의 명령어들로 구성됩니다. 명령어는 세미콜론(";")으로 끝나는 토큰들의 시퀀스로 구성됩니다. 입력 스트림의 끝 또한 명령어를 종료합니다. 어떤 토큰이 유효한지는 특정 명령어의 구문에 따라 달라집니다.

토큰은 키워드, 식별자, 따옴표로 묶인 식별자, 리터럴(또는 상수), 또는 특수 문자 기호가 될 수 있습니다. 토큰들은 일반적으로 공백(스페이스, 탭, 줄바꿈)으로 구분되지만, 모호성이 없는 경우(일반적으로 특수 문자가 다른 토큰 유형과 인접한 경우에만 해당)에는 반드시 그럴 필요는 없습니다.

예를 들어, 다음은 (구문적으로) 유효한 SQL 입력입니다:

```sql
SELECT * FROM MY_TABLE;
UPDATE MY_TABLE SET A = 5;
INSERT INTO MY_TABLE VALUES (3, 'hi there');
이는 한 줄에 하나씩, 세 개의 명령어 시퀀스입니다(이는 필수는 아닙니다; 한 줄에 둘 이상의 명령어가 있을 수 있고, 명령어는 여러 줄에 걸쳐 유용하게 분할될 수 있습니다).
```

또한, SQL 입력에는 주석이 포함될 수 있습니다. 주석은 토큰이 아니며, 실질적으로 공백과 동등합니다.

SQL 구문은 어떤 토큰이 명령어를 식별하고 어떤 것이 피연산자나 매개변수인지에 대해 매우 일관적이지 않습니다. 처음 몇 개의 토큰은 일반적으로 명령어 이름이므로, 위의 예시에서는 보통 "SELECT", "UPDATE", "INSERT" 명령어라고 말합니다. 하지만 예를 들어 UPDATE 명령어는 항상 특정 위치에 SET 토큰이 나타나야 하며, 이 특정한 형태의 INSERT도 완성되기 위해서는 VALUES가 필요합니다. 각 명령어에 대한 정확한 구문 규칙은 제6부에서 설명됩니다.
